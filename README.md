# 五子棋强化学习项目

基于深度强化学习(DQN)的五子棋AI训练与人机对战系统。

## 项目结构

```
RL/
├── gomoku_env.py      # 五子棋游戏环境
├── dqn_agent.py       # DQN智能体实现
├── train.py           # 训练脚本
├── play.py            # 人机对战脚本
├── requirements.txt   # 依赖包
└── models/            # 模型保存目录（自动创建）
```

## 增强学习学习路径

### 第一阶段：基础理论（1-2周）
1. **马尔可夫决策过程(MDP)**
   - 状态(State): 棋盘配置
   - 动作(Action): 在空位落子
   - 奖励(Reward): 胜+1，负-1，平0
   - 转移概率: 确定性环境

2. **价值函数**
   - Q(s,a): 在状态s执行动作a的期望回报
   - 通过神经网络近似Q函数

3. **策略**
   - ε-贪婪策略: 平衡探索与利用
   - 策略改进: 通过训练逐步提升

### 第二阶段：深度强化学习（2-3周）
1. **DQN核心技术**
   - 经验回放(Experience Replay): 打破数据相关性
   - 目标网络(Target Network): 稳定训练
   - 卷积神经网络: 提取棋盘特征

2. **训练流程**
   - 自我对弈收集经验
   - 从经验池采样训练
   - 定期更新目标网络

### 第三阶段：项目实战（3-4周）
1. **环境设计** - `gomoku_env.py`
   - 15x15标准棋盘
   - 五子连珠判定
   - 状态表示: 3通道(黑棋、白棋、当前玩家)

2. **智能体训练** - `train.py`
   - 自我对弈生成数据
   - DQN网络优化
   - 训练曲线可视化

3. **人机对战** - `play.py`
   - 与训练好的AI下棋
   - AI自我对弈展示

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 训练AI

```bash
python train.py
```

训练参数可在`train.py`中调整：
- `num_episodes`: 训练局数(默认5000)
- `board_size`: 棋盘大小(默认15)
- `save_interval`: 模型保存间隔(默认500局)

### 3. 人机对战

```bash
python play.py
```

选择模式：
1. 人机对战：与AI下棋
2. AI自我对弈展示：观看AI对弈

## 核心概念解析

### 1. 状态表示
棋盘状态用3个通道表示：
- 通道0: 黑棋位置(1表示有黑棋，0表示无)
- 通道1: 白棋位置(1表示有白棋，0表示无)
- 通道2: 当前玩家(全1或全-1)

### 2. 奖励设计
- 获胜: +1.0
- 失败: -1.0
- 平局: 0.0
- 中间步骤: 0.0
- 非法动作: -10.0(立即结束)

### 3. 网络结构
```
输入(15x15x3) 
→ Conv2D(64) 
→ Conv2D(128) 
→ Conv2D(128) 
→ FC(512) 
→ FC(225) 
→ 输出Q值
```

### 4. 训练技巧
- **经验回放**: 缓冲区容量10000
- **目标网络**: 每100步更新
- **ε-贪婪**: ε从1.0衰减到0.01
- **批次大小**: 64
- **学习率**: 0.001
- **折扣因子**: 0.99

   - 与人类对战表现

## 常见问题

**Q: 训练需要多久？**
A: 在CPU上约2-3小时(5000局)，GPU会快很多。

**Q: AI不够强怎么办？**
A: 增加训练局数、调大网络、改进奖励函数。

**Q: 如何提高训练速度？**
A: 使用GPU、减小棋盘尺寸(如9x9)、并行训练。

**Q: 能否实现其他棋类？**
A: 可以！修改环境规则即可适配象棋、围棋等。
